## 第一阶段：Demo 补强（现在 → 4月底）

### 目标：让 Demo 看起来"不只是 demo"

**① 产品维度（MiniMax Claw 内）**

| 模块 | 现在状态 | 目标状态 |
|------|---------|---------|
| 名称 | China Mate | ✅ VisitCN |
| 欢迎语 | 基础 | 完整+有吸引力 |

| 材料 | 状态 | 行动 |

**③ 本周任务清单**

```
□ 1. 注册公司（拿到营业执照）

□ 7. 确定推荐人（投资人/学者/行业专家）
```

## 第二阶段：U30 申报冲刺（5月）

### 目标：完整提交申报材料

**申报材料最终清单**

```
【必须提交】
□ 1. 个人基本信息（身份证、学历、职位）
□ 2. 公司营业执照
□ 3. 项目介绍（500-800字）
   - 痛点
   - 解决方案
   - 产品介绍
   - 商业模式
   - 团队
□ 4. 创始人故事（500字）
□ 5. 核心数据
   - 目标市场规模
   - 已有进展（哪怕是demo阶段）
□ 6. 推荐信 × 1-2封

【加分项】
□ 7. 产品演示视频（1分钟）
□ 8. 媒体报道截图
□ 9. 博主合作MOU
□ 10. 行程演示案例
```

**5月时间线**

| 周 | 任务 |
|----|------|
| 5月第1周 | 完善创始人故事 + 项目介绍 |
| 5月第2周 | 收集所有材料 + 润色 |
| 5月第3周 | 推荐信到位 + 材料整合 |
| 5月第4周 | 最终检查 + 提交 |

## 第三阶段：U30 结果 → 融资（6-12月）

### 目标：借助 U30 背书拿到天使轮

```
U30 通常在10-12月发布

发布前（6-9月）：
- 持续迭代产品
- 积累真实用户数据
- 参加创业比赛（制造曝光）
- 接触天使投资人

发布后（10-12月）：
- U30 榜单背书 → 投资人信任度大增
- 启动正式融资路演
- 目标：天使轮 300-500万估值/融资

如果没上：
- 不代表失败，继续迭代
- 可以次年再申报（有实际运营数据后更有优势）
```

## 第四阶段：APP 正式开发（获得融资后）

### 产品路线图

```
【APP 1.0】（MVP，3-6个月开发）
核心功能：
1. AI行程生成（继承 demo 能力）
2. 用户账号体系
3. 行程收藏/导出
4. 基础信息查询（签证、交通）

技术方案：
- 前端：Flutter（iOS+Android）
- 后端：Node.js / Python FastAPI
- AI：MiniMax API
- 数据库：PostgreSQL

【APP 2.0】（6-12个月）
新增功能：
1. 酒店/机票预订接入（API合作）
2. 实时景点预约
3. 社区功能（用户分享）
4. 博主内容专区

【APP 3.0】（12个月+）
新增功能：
1. 行程分享/打卡
2. 行程中实时辅助（AR翻译等）
3. 跨境支付深度集成
4. 企业差旅版
```

### 技术团队组建

| 阶段 | 团队规模 | 角色 |
|------|---------|------|
| MVP阶段 | 2-3人 | 前端+后端+AI |
| 正式运营 | 5-8人 | +产品+运营+市场 |
| 扩张期 | 10人+ | +技术负责人+BD |

## 第五阶段：增长与规模化

### 用户增长策略

```
【冷启动（0→1000用户）】
1. 境外旅行博主合作（核心渠道）
2. 小红书/抖音"外国博主在中国"系列
3. 旅行论坛/Reddit/Quora 推广
4. 景区合作引流

【增长期（1000→10万用户）】
1. SEO + 内容营销（英文旅游博客）
2. KOL 合作（头部旅游博主）
3. 平台合作（Trip.com、Booking等导流）
4. UGC 内容社区

【规模化（10万+用户）】
1. 品牌广告
2. 异业合作（航空公司、银行卡组织）
3. 商务差旅企业客户
```

### 收入目标

| 阶段 | 时间 | 目标月流水 |
|------|------|-----------|
| MVP验证 | 0-6个月 | — |
| 商业化起步 | 6-12个月 | ¥10-50万 |
| 增长期 | 12-24个月 | ¥100-500万 |
| 规模化 | 24个月+ | ¥1000万+ |

## 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| U30 申报未通过 | 中 | 次年再申报，有实际数据后更有优势 |
| 天使融资不顺 | 中 | 先用自有资金撑，做收入 |
| APP 开发延期 | 高 | 先上 H5/小程序，敏捷验证 |
| 竞争加剧 | 中 | 专注AI行程生成，建立技术壁垒 |
| 入境游政策变化 | 低 | 平台定位是"工具"，不依赖特定政策 |
