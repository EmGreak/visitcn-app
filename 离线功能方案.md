# VisitCN 离线功能方案

> 版本：V1.0
> 更新：2026-05-13
> 状态：方案设计

---

## 一、方案概述

参考VOYO的离线功能设计，VisitCN需要提供离线内容包，让外国游客在网络不稳定的环境下仍能使用核心功能。

---

## 二、离线功能需求分析

### 2.1 用户场景

| 场景 | 网络状态 | 需求 |
|------|----------|------|
| 地铁/地下 | 无信号 | 地图、路线离线查看 |
| 偏远地区 | 信号弱 | 紧急信息、基础指南 |
| 飞机上 | 关机 | 完整城市指南阅读 |
| 网络故障 | 断网 | 支付指南、翻译短语 |

### 2.2 离线内容优先级

| 优先级 | 内容 | 大小估算 | 说明 |
|--------|------|----------|------|
| P0 | 城市基础信息 | 2MB | 紧急电话、地标、交通 |
| P0 | Visa信息 | 500KB | 签证类型、申请流程 |
| P0 | 支付指南 | 500KB | Alipay/WeChat使用说明 |
| P1 | 地图数据 | 10-30MB/城市 | 离线地图切片 |
| P1 | 常用短语 | 1MB | 中英对照短语 |
| P2 | 城市攻略 | 5MB/城市 | 美食、景点详细指南 |
| P2 | AI助手离线版 | - | 预置问答库 |

## 三、技术方案

### 3.1 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                    VisitCN App                          │
├─────────────────────────────────────────────────────────┤
│  Online Mode                    Offline Mode            │
│  ├── Full AI Assistant         ├── Preloaded Q&A      │
│  ├── Live Maps                 ├── Cached Tiles        │
│  ├── Real-time Data            ├── Static Content      │
│  └── API-driven               └── Local Storage        │
├─────────────────────────────────────────────────────────┤
│              Offline Content Manager                    │
│  ├── Download Manager                                 │
│  ├── Storage Manager (Hive/SQLite)                    │
│  └── Sync Controller                                   │
└─────────────────────────────────────────────────────────┘
```

### 3.2 技术选型

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 本地存储 | Hive / SQLite | 轻量级键值数据库 |
| 地图离线 | Flutter Map + Tile Caching | OpenStreetMap方案 |
| 内容包 | JSON/ZIP | 结构化数据压缩包 |
| 下载管理 | dio + flutter_downloader | 断点续传 |
| 状态管理 | Provider / Riverpod | 离线状态同步 |

### 3.3 离线存储结构

```
/offline_data/
├── manifest.json          # 离线包索引
├── cities/
│   ├── shanghai/
│   │   ├── info.json     # 城市基本信息
│   │   ├── visa.json     # 签证信息
│   │   ├── transport.json # 交通指南
│   │   ├── phrases.json  # 常用短语
│   │   └── map_tiles/    # 地图切片
│   ├── beijing/
│   └── ...
├── common/
│   ├── emergency.json    # 紧急信息（通用）
│   ├── payment.json      # 支付指南
│   └── phrases_common.json # 通用短语
```

## 四、功能模块详细设计

### 4.1 离线内容包下载

**下载管理器功能：**
- 显示可用城市列表
- 显示已下载/未下载状态
- 显示大小和下载进度
- 断点续传支持
- 自动更新检测

**下载触发时机：**
1. 用户主动下载（设置页面）
2. 首次进入城市时提示下载
3. WiFi环境下自动下载常用城市

### 4.2 离线地图

**方案A：OpenStreetMap + 离线切片**
- 使用 `flutter_map` + `flutter_map_tile_caching`
- 预下载重点区域（如外滩、故宫）
- 按兴趣点（POI）下载

**方案B：集成第三方离线地图SDK**
- 考虑：Mapsforge（开源方案）
- 或：使用高德/百度地图的离线API（如有）

**推荐：方案A**（成本低、可控性强）

### 4.3 离线AI助手

**简化版离线问答库：**

预置50-100个高频问题：
- "How to take subway?"
- "Where is the nearest hospital?"
- "How to use Alipay?"
- "Emergency number"
- "How to get to airport?"

数据格式：
```json
{
  "questions": [
    {
      "q": "how to use alipay",
      "keywords": ["alipay", "payment", "pay"],
      "a": "Alipay is China's main mobile payment app...",
      "image": "alipay_guide.png"
    }
  ]
}
```

### 4.4 离线支付指南

**内容：**
- Alipay使用步骤（截图+文字）
- WeChat Pay说明（限制说明）
- 现金使用场景
- 信用卡使用指南

**离线存储：**
- 静态HTML或Markdown
- 内嵌图片（Base64或本地文件）

### 4.5 离线紧急信息

**通用紧急信息：**
- 中国国际紧急电话：12308
- 当地急救电话：120
- 报警电话：110
- 消防电话：119
- 大使馆联系方式

**城市特定：**
- 当地医院列表（含地址、电话）
- 警察局地址
- 最近的药房

## 五、数据同步策略

### 5.1 增量更新

**首次下载：** 下载完整离线包
**后续更新：** 只下载差异部分

```json
{
  "version": "1.0.1",
  "last_updated": "2026-05-13",
  "changelog": [
    "Updated visa requirements for L visa",
    "Added new restaurant recommendations"
  ],
  "download_url": "https://cdn.visitcn.app/offline/shanghai_v1.0.1.zip"
}
```

### 5.2 同步时机

| 时机 | 行为 |
|------|------|
| WiFi + 充电中 | 自动下载已选城市的更新 |
| 流量环境 | 仅同步紧急更新 |
| 用户手动触发 | 完全同步 |

## 八、技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 离线包过大 | 用户不愿下载 | 分城市、分模块下载 |
| 地图切片更新 | 存储浪费 | 增量更新机制 |
| 内容过期 | 信息不准 | 自动更新检测 + 提醒 |
| 存储空间不足 | 无法下载 | 存储管理 + 清理提示 |
