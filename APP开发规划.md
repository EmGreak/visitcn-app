# VisitCN App 开发规划

> 版本：v1.0 | 创建：2026-04-10
> 目标：iOS + Android 双平台 App

---

## 一、技术方案

### 推荐方案：Flutter 跨平台

| 对比项 | Flutter ✅ | React Native | 原生开发（iOS+Android分别写） |
|--------|-----------|--------------|--------------------------|
| 开发效率 | 1套代码跑双平台 | 1套代码跑双平台 | 2套代码 |
| 开发成本 | 中 | 中 | 高 |
| 性能 | 接近原生 | 较好 | 最优 |
| 学习曲线 | 需要Flutter基础 | 需要JS基础 | 各需要精通 |
| 适合阶段 | MVP→正式产品 | MVP→正式产品 | 不推荐 |

**结论**：选 Flutter，效率最高。

### 核心技术栈

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | Flutter 3.x | iOS + Android 双平台 |
| AI 对话 | MiniMax M2.7 API | 已有 Plus key，可直接用 |
| 语音合成（TTS） | MiniMax Speech 2.8 API | 出境游场景刚需 |
| 语音识别（STT） | MiniMax API | 用户语音输入 |
| 后端 | Firebase / Supabase | 认证+数据库+存储 |
| 状态管理 | Riverpod / BLoC | Flutter 官方推荐 |
| 地图 | 高德地图 / Google Maps | 境外用 Google，境内用高德 |

## 二、MiniMax 全模态能力在 App 中的应用

| 能力 | 应用场景 |
|------|---------|
| **MiniMax M2.7**（文本） | AI 行程生成、问答、推荐 |
| **Speech 2.8**（TTS） | 行程语音播报、多语言朗读 |
| **语音识别** | 用户语音输入（英文/日文/韩文直接说） |
| **image-01**（生图） | 生成个性化城市风景图/旅行卡片 |
| **Music-2.6** | 生成旅行氛围音乐（可选） |
| **Hailuo-2.3**（视频） | 景点预览视频（进阶功能） |

## 三、功能规划

### MVP 版本（核心功能，2-4个月）

优先级从高到低：

| 优先级 | 功能 | 说明 |
|-------|------|------|
| P0 | AI 对话界面 | 继承 Web Demo 的核心对话能力 |
| P0 | 行程生成结果展示 | 美观的每日行程卡片 |
| P0 | 多语言支持 | 英文为主，中/日/韩/泰/西/法为辅 |
| P1 | 语音输入/输出 | 用户说话生成行程，行程可朗读 |
| P1 | 用户反馈 | 对行程满意度评分 |
| P1 | 收藏/分享 | 行程可分享到 WhatsApp/Instagram |
| P2 | 登录/注册 | 手机号/Google/Apple 登录 |
| P2 | 热门目的地展示 | 城市/景点首页推荐 |
| P3 | 支付功能 | 接入微信/支付宝国际版 |

### 正式版（6-12个月）

| 功能 | 说明 |
|------|------|
| 博主内容专区 | 旅行博主入驻，发布攻略 |
| 实时景点导览 | AR 翻译 + 语音讲解 |
| 酒店/机票预订 | 接入 API（Booking/Trip.com） |
| 社区功能 | 用户分享经验 |
| 企业差旅版 | To B 版本 |

## 五、开发里程碑

### 阶段一：准备（1-2周）

```
□ 注册 Apple Developer 账号（iOS需要，99美元/年）
□ 注册 Google Play 开发者账号（25美元，一次性）
□ 注册 Flutter 环境
□ 搭建项目架构
□ 注册 Firebase 项目（后端服务）
□ 申请高德地图 API Key
```

### 阶段二：核心开发（6-10周）

| 周数 | 任务 |
|------|------|
| 第1-2周 | Flutter 项目初始化 + 基础架构 |
| 第3-4周 | AI 对话界面开发（MiniMax API 集成） |
| 第5-6周 | 行程生成 + 展示页面 |
| 第7-8周 | 语音输入/输出集成 |
| 第9-10周 | 收藏/分享 + 用户反馈功能 |

### 阶段三：内测（2-3周）

```
□ 邀请 10-20 名境外用户内测
□ 收集反馈
□ 修复 Bug
□ 优化体验
```

### 阶段四：上架（1-2周）

```
□ App Store 上架（审核约1-3天）
□ Google Play 上架（审核约1-7天）
□ 准备应用商店素材（截图/描述/隐私政策）
```

## 八、iOS vs Android 优先级

| 对比 | iOS | Android |
|------|-----|---------|
| 开发 | 审核更严 | 审核较松 |
| 用户质量 | 高消费力（欧美用户多） | 东南亚/日韩用户多 |
| 收入潜力 | 高 | 中 |
| 上架难度 | 较难 | 较易 |

**建议**：两个版本并行开发，Flutter 一起出。如果非要选一个先上：**iOS优先**（欧美用户消费力更强，与免签政策利好匹配）。

## 十一、关于 H5/小程序过渡方案

如果你觉得 App 开发周期太长，可以先做 **H5 版本**，好处：

- 微信/小红书内直接打开，不用下载
- 开发更快（1-2周）
- 便于快速验证用户

微信小程序可以申请"外国人来华"相关类目，也符合政策方向。
